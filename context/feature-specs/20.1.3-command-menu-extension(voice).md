## /voice — Speech-to-Text Input

Adds a `/voice` command that **toggles voice input mode** on or off. When enabled, the user can **hold Space** (long press) to record and release to transcribe. The transcript is appended at the cursor position in the textarea. A short Space press still inserts a normal space — voice mode does not break typing.

This mirrors how Claude Code handles `/voice`: a persistent toggle stored in config, not a one-shot action.

---

### Philosophy

KOINCODE is free and open-source. Voice input should be too. Transcription is handled by **Whisper running locally via `@xenova/transformers`** — a pure JavaScript + WASM package that ships as a regular npm dependency. No native binary, no system tool, no API key required for transcription. Models are downloaded automatically to a local cache on first use.

---

### Transcription: `@xenova/transformers`

This is the core architectural decision. Instead of requiring users to install `whisper.cpp` (a C++ binary), we use the `@xenova/transformers` package which runs Whisper in pure JavaScript + WASM inside the Node.js/Bun process.

**Why this approach:**
- Zero system dependencies for transcription — installs with the app via npm/bun
- Works on macOS, Linux, and Windows without any extra setup
- Models download automatically to `~/.koincode/whisper/` on first use
- Slower than native whisper.cpp (~2–4x) but for 5–15 second clips this is acceptable
- Same underlying Whisper model, same accuracy

**Startup cost:** loading the WASM runtime and model into memory takes 2–5 seconds — but this happens only **once per process lifetime** (i.e. once per time the user opens the app), not once per chat session. KOINCODE is a long-running process; navigating between chat conversations does not restart it. The pipeline instance is cached at module scope (a file-level variable that persists for the entire app session), so every subsequent transcription within the same app run is fast. The cold start is only noticeable the very first time the user presses Space to record after launching the app.

**Model sizes** (user-configurable via `whisperModel` in `~/.koincode/config.json`):
| Name | Size | Speed | Accuracy |
|------|------|-------|----------|
| `tiny` | 75MB | Fastest | Good for clear speech |
| `base` | 140MB | Fast | **Default** — good balance |
| `small` | 460MB | Moderate | Near-API quality |

Each model is downloaded once and cached permanently. Switching sizes only downloads the new model the first time — previously cached models are instant.

**Model format note:** The raw whisper.cpp format uses `.bin` files named `ggml-base.bin` etc. (`ggml` is a C tensor library by the same author). `@xenova/transformers` uses its own ONNX-based format and manages all of this internally — the implementation never touches `.bin` files or whisper.cpp.

### Backend priority

```
1. OpenAI Whisper API        → if openaiKey is set in config (fast, no cold start, ~$0.006/min)
2. @xenova/transformers      → if no OpenAI key (free, local, bundled, works offline)
```

The user is already spending money on AI calls when they have an OpenAI key, so using the Whisper API as the primary backend when available gives a better UX: instant transcription with no cold start and no memory overhead. At ~$0.006/minute it costs essentially nothing.

Without an OpenAI key, `@xenova/transformers` provides the same Whisper model running locally for free — no API key, no internet required after the model download.

**Override:** users who prefer local-only (privacy, offline use) can set `whisperBackend: "local"` in `~/.koincode/config.json` to force xenova regardless of whether an OpenAI key is present. The default is `"auto"`.

The switch between backends is silent — the user sees the same transcription flow either way.

---

### Audio recording: platform-native

Transcription is fully bundled. Recording still requires a system tool, but uses whatever is already present on the platform:

| Platform | Tool | Status |
|----------|------|--------|
| Linux | `arecord` | Pre-installed on most distros (ALSA utils) |
| macOS | `sox` | Must be installed: `brew install sox` |
| Windows | `powershell.exe` | Built-in — spawned as a subprocess regardless of what shell the user is in |

**Windows note:** We call `powershell.exe` directly as a subprocess from Node.js/Bun. It doesn't matter whether the user's terminal is CMD, Git Bash, VS Code terminal, or anything else — we invoke the PowerShell binary by absolute path independently of the user's current shell.

**macOS note:** There is no built-in CLI audio recorder on macOS (`afplay` plays audio but there is no `afrecord` equivalent). `sox` is the correct tool. The app shows a clear install message (`brew install sox`) if it is missing.

On first use of `/voice`, if the recording tool is missing the app shows a platform-specific install hint and aborts — it does not crash.

### User flow

1. User types `/voice` in the command menu — voice mode toggles. `voiceInput: true` written to `~/.koincode/config.json`. Toast: `"Voice input enabled — hold space to speak"`.
2. The status bar shows a `voice` indicator while voice mode is active.
3. User **holds Space** for more than ~300ms → recording starts. Placeholder changes to `"Recording… release space to stop"`.
4. User **releases Space** → recording stops. Placeholder shows `"Transcribing…"` briefly.
5. Transcript is **appended at the cursor position** in the textarea (textarea content is preserved; the user can have existing text).
6. A trailing space is added after the transcript so the user can keep typing immediately.
7. `/voice` again → `voiceInput: false`, indicator disappears, Space returns to normal.

---

### Space key behaviour

The Space key is intercepted only when voice mode is enabled. The distinction between a short press (normal space) and a long press (record) is made with a ~300ms timer:

```
Space keydown → start 300ms timer
  If keyup fires before timer:  insert normal space
  If timer fires before keyup:  begin recording immediately
    Space keyup → stop recording → transcribe → append at cursor
```

This means the user can type normally with spaces even when voice mode is on. Recording only triggers on a deliberate hold. Textarea content is irrelevant — recording works regardless of what's in the input, and the transcript appends at wherever the cursor currently is.

#### Step 2 — Transcription module

**`packages/cli/src/lib/whisper.ts`** — new module:

```ts
export async function transcribe(audioPath: string, config: VoiceConfig): Promise<string>;
// Selects backend based on config.whisperBackend:
//   "auto"   → OpenAI if config.openaiKey present, else local
//   "openai" → calls openai.audio.transcriptions.create (whisper-1)
//   "local"  → runs @xenova/transformers pipeline
// Cleans up the temp audio file after transcription
// Returns transcript string (empty string on failure — caller shows toast)
```

**Module-scope pipeline cache (local backend only):**

```ts
// whisper.ts — top-level variable, persists for the entire process lifetime
let _pipeline: AutomaticSpeechRecognitionPipeline | null = null;

async function getLocalPipeline(model: string) {
  if (!_pipeline) {
    // Runs once per app launch — 2–5s cold start on first voice use
    _pipeline = await pipeline("automatic-speech-recognition", `Xenova/whisper-${model}`);
  }
  return _pipeline;
}
```

`_pipeline` is a file-level variable. Every call to `transcribe()` across all chat sessions within the same app run reuses the same loaded model. The cold start only occurs the first time the user records audio after opening the app. Subsequent uses — including switching between chat sessions — are instant because the process never restarts.

---

#### Step 3 — Audio recorder module

**`packages/cli/src/lib/voice-recorder.ts`** — new module:

```ts
export async function checkRecorderAvailable(): Promise<{
  ok: boolean;
  hint?: string; // platform-specific install instruction if missing
}>;

export type RecorderHandle = {
  stop: () => Promise<string>; // resolves to temp .wav file path
};

export async function startRecording(): Promise<RecorderHandle>;
// Generates a temp path: os.tmpdir()/koincode-voice-<timestamp>.wav
// Spawns platform recorder:
//   linux:  arecord -f S16_LE -r 16000 -c 1 <path>
//   darwin: sox -d -r 16000 -c 1 -b 16 <path>
//   win32:  powershell.exe -Command "<audio capture script>" <path>
// Returns handle whose .stop() kills the process and returns the wav path
```

Output is always 16kHz mono WAV — the format Whisper expects.

---

#### Step 4 — Voice state in InputBar

**`packages/cli/src/components/input-bar.tsx`**:

```ts
const [voiceState, setVoiceState] = useState<"idle" | "recording" | "transcribing">("idle");
const recorderRef = useRef<RecorderHandle | null>(null);
const spaceTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);
```

**Space keydown handler** (only when `voiceInput === true`):
```ts
// Start 300ms timer; if still held after threshold, begin recording
spaceTimerRef.current = setTimeout(async () => {
  const { ok, hint } = await checkRecorderAvailable();
  if (!ok) {
    toast.show({ variant: "error", message: hint ?? "Recorder not available" });
    return;
  }
  recorderRef.current = await startRecording();
  setVoiceState("recording");
}, 300);
```

**Space keyup handler** (only when `voiceInput === true`):
```ts
if (spaceTimerRef.current) {
  clearTimeout(spaceTimerRef.current);
  spaceTimerRef.current = null;

  if (voiceState === "idle") {
    // Short press — insert normal space
    textarea.insertText(" ");
    return;
  }
}

if (voiceState === "recording" && recorderRef.current) {
  const wavPath = await recorderRef.current.stop();
  setVoiceState("transcribing");
  const text = await transcribe(wavPath, config.whisperModel);
  if (text) textarea.insertText(text + " ");
  setVoiceState("idle");
}
```

**Textarea placeholder** by `voiceState`:
- `idle` + voice enabled → `"Hold space to speak…"`
- `recording` → `"Recording… release space to stop"`
- `transcribing` → `"Transcribing…"`
- `idle` + voice disabled → normal placeholder

### Decisions

- **OpenAI Whisper API is the default when an OpenAI key is present.** Instant transcription, no cold start, no memory overhead. Cost is negligible (~$0.006/min). Users already paying for AI calls via their key.
- **`@xenova/transformers` is the default when no OpenAI key is present.** Zero system dependencies, ships with the app via npm, works offline. Cold start (2–5s) occurs once per process lifetime — once per app launch, not once per chat session. Subsequent transcriptions within the same app run are instant.
- **`whisperBackend` config field** lets users override the automatic selection: `"auto"` (default), `"openai"`, or `"local"`.
- **Recording still needs a platform tool.** Linux uses pre-installed `arecord`; macOS requires `sox` (one `brew install`); Windows uses `powershell.exe` (built-in, spawned as a subprocess regardless of the user's current shell).
- **`macOS has no built-in CLI audio recorder.`** `afplay` exists but only plays audio. `afrecord` does not exist. `sox` is the correct tool.
- **Long press = record, short press = normal space.** 300ms threshold. Textarea content is irrelevant. Transcript appends at cursor position with a trailing space.
- **System dictation ruled out.** macOS and Windows built-in dictation degrades significantly on technical vocabulary — function names, identifiers, API terms. Whisper handles this correctly even on the `tiny` model. Not worth the accessibility permission complexity.
- **Model cache is permanent.** Each downloaded model size lives in `~/.koincode/whisper/`. Switching sizes only downloads the new model once.
