## /handoff ‚Äî Continue in a New Session with Context

Adds a `/handoff` command that generates a concise summary of the current session, creates a new session pre-seeded with that summary, and navigates there. The user lands in a fresh session where the AI already knows what was accomplished ‚Äî like passing a baton.

The original session is untouched. The new session's first message is a synthetic summary that gives the AI full context without the full message history.

---

### User flow

1. User types `/handoff` and selects it from the command menu
2. A brief "Summarizing session‚Ä¶" toast or status indicator appears
3. The server generates a summary of the current session (single LLM call, no streaming needed)
4. A new session is created with the summary injected as an opening assistant message
5. The CLI navigates to the new session ‚Äî the input bar is focused and ready
6. The AI sees the summary as context on the first user message

---

### How the summary is injected

The new session's `messages` array is pre-populated with a synthetic exchange:

```json
[
  {
    "role": "user",
    "content": "Here is a summary of work completed in a previous session. Use this as your starting context."
  },
  {
    "role": "assistant",
    "content": "<generated summary>"
  }
]
```

This is the cleanest approach: no special message types, no schema changes, works with the existing LLM history pipeline unchanged.

---

### Implementation steps

#### Step 1 ‚Äî Server: handoff endpoint

**`packages/server/src/routes/sessions.ts`** ‚Äî new route:

```
POST /sessions/:id/handoff
```

Steps inside the handler:
1. Load the source session's messages
2. Call the LLM (non-streaming, single turn) with a system prompt asking for a summary:
   ```
   Summarize the work done in this conversation concisely. Focus on:
   - What was built or changed
   - Decisions made and why
   - Any open questions or next steps
   Keep it under 300 words. Write in second person ("You were working on‚Ä¶").
   ```
3. Create a new session with:
   - `title`: `"Continued: <source session title>"`
   - `messages`: the synthetic summary exchange (see above)
4. Return `{ sessionId: string }` ‚Äî the new session's ID

The summary LLM call should use the same model that was active in the source session (stored in the last assistant message's metadata, or fall back to the server default).

---

#### Step 2 ‚Äî CLI: API client

**`packages/cli/src/lib/api-client.ts`** ‚Äî expose `POST /sessions/:id/handoff` returning `{ sessionId: string }`.

---

#### Step 3 ‚Äî CLI: command wiring

**`packages/cli/src/components/command-menu/types.ts`** ‚Äî add to `CommandContext`:

```ts

```

**`packages/cli/src/components/command-menu/commands.tsx`** ‚Äî add command:

```ts
{
  name: "handoff",
  description: "Summarize this session and continue in a new one",
  value: "/handoff",
  action: async (ctx) => {
    await ctx.handoff();
  },
},
```

**`packages/cli/src/components/input-bar.tsx`** ‚Äî include `handoff` in the `CommandContext` object:

```ts
handoff: onHandoff ?? (() => Promise.resolve()),
```

**`packages/cli/src/components/session-shell.tsx`** ‚Äî thread `onHandoff?: () => Promise<void>` prop to `InputBar`.

---

#### Step 4 ‚Äî CLI: session screen

**`packages/cli/src/screens/session.tsx`** ‚Äî implement `handleHandoff` in `SessionChat`:

```ts
const handleHandoff = async () => {
  toast.show({ variant: "info", message: "Summarizing session‚Ä¶" });
  try {
    const res = await apiClient.sessions[":id"].handoff.$post({
      param: { id: session.id },
    });
    if (!res.ok) throw new Error(await getErrorMessage(res));
    const { sessionId } = await res.json();
    navigate(`/session/${sessionId}`);
  } catch (err) {
    toast.show({
      variant: "error",
      message: err instanceof Error ? err.message : "Handoff failed",
    });
  }
};
```

Pass `handleHandoff` as `onHandoff` down through `SessionShell` ‚Üí `InputBar`.

### Decisions

- **No confirmation step.** Navigate immediately. The summary is visible as the first message in the new session ‚Äî if the user dislikes it they can keep going from there.
- **Handoff from a near-empty session** (fewer than 2 exchanges): skip the LLM summarization step and just create a plain new session, same as `/new`.
