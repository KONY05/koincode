## /clear ‚Äî Reset Conversation Context

Adds a `/clear` command that resets the AI's context window without deleting the session or its messages. Messages sent after `/clear` start a fresh context ‚Äî the model no longer sees anything from before the boundary. All messages (before and after) remain in the database and visible history is also hidden post-clear.

This matches how Claude Code implements `/clear`: the session continues under the same ID, the full message history stays on disk, but subsequent LLM turns only receive messages from the clear point onward.

---

### How the boundary is stored

No schema migration needed. A sentinel object is appended to the `messages` JSON array on the Session:

```json
{ "type": "clear_boundary", "clearedAt": "<ISO timestamp>" }
```

When the server builds LLM history for a chat request, it scans the messages array for the **last** `clear_boundary` marker and slices from that index onward. Everything before the boundary is ignored by the model.

The UI does the same slice for rendering ‚Äî only messages after the last boundary are shown.

### Implementation steps

#### Step 1 ‚Äî Shared type

**`packages/shared/src/schemas.ts`** ‚Äî add a `ClearBoundaryMarker` type to the message union so both server and CLI can identify the marker:

```ts
export type ClearBoundaryMarker = {
  type: "clear_boundary";
  clearedAt: string;
};
```

---

#### Step 2 ‚Äî Server: clear endpoint

**`packages/server/src/routes/sessions.ts`** ‚Äî new route:

```
POST /sessions/:id/clear
```

- Load the session
- Append `{ type: "clear_boundary", clearedAt: new Date().toISOString() }` to `session.messages`
- Save and return `{ clearedAt: string }`

---

#### Step 3 ‚Äî Server: filter messages by boundary

**`packages/server/src/routes/chat.ts`** ‚Äî in the section that loads session messages and builds the LLM history:

```ts
function getMessagesAfterLastClear(messages: unknown[]): Message[] {
  let lastClearIdx = -1;
  for (let i = messages.length - 1; i >= 0; i--) {
    if ((messages[i] as any)?.type === "clear_boundary") {
      lastClearIdx = i;
      break;
    }
  }
  return (messages.slice(lastClearIdx + 1) as Message[]).filter(
    (m) => m.type !== "clear_boundary"
  );
}
```

Use this instead of passing the raw messages array to the AI SDK.

**`packages/cli/src/lib/api-client.ts`** (or wherever the typed Hono client is defined) ‚Äî expose the new `POST /sessions/:id/clear` endpoint so the CLI can call it.

#### Step 5 ‚Äî CLI: command wiring

**`packages/cli/src/components/command-menu/types.ts`** ‚Äî add to `CommandContext`:

```ts
clearSession: () => Promise<void>;
```

**`packages/cli/src/components/command-menu/commands.tsx`** ‚Äî add command:

```ts
{
  name: "clear",
  description: "Reset AI context ‚Äî start fresh without losing history",
  value: "/clear",
  action: async (ctx) => {
    await ctx.clearSession();
  },
},
```

**`packages/cli/src/components/input-bar.tsx`** ‚Äî include `clearSession` in the `CommandContext` object passed to `command.action(...)`:

```ts
clearSession: onClearSession ?? (() => Promise.resolve()),
```

**`packages/cli/src/components/session-shell.tsx`** ‚Äî thread `onClearSession?: () => Promise<void>` prop through to `InputBar`.

---

#### Step 6 ‚Äî CLI: session screen

**`packages/cli/src/screens/session.tsx`** ‚Äî in `SessionChat`:

- Derive `clearBoundaryIndex` from `messages`: find the index of the last `clear_boundary` marker (scan `useChat`'s messages or the raw session messages)
- In the transcript render, slice from `clearBoundaryIndex`
- Implement `handleClearSession`:
  1. Call `POST /sessions/:id/clear` via the API client
  2. Append the marker to local state so the UI updates immediately without a full refetch

```ts
const [localClearIdx, setLocalClearIdx] = useState(() =>
  findLastClearBoundary(initialMessages)
);

const handleClearSession = async () => {
  await apiClient.sessions[":id"].clear.$post({ param: { id: session.id } });
  setLocalClearIdx(messages.length); // hide everything currently visible
};
```

The transcript render becomes:

```tsx
{transcript.slice(localClearIdx).map(...)}
```
