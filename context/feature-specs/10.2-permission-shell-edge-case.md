# Shell Command Edge Cases & Security Improvements

The user raised a valid point: while the command binary (e.g. `cat`, `echo`, `rm`) might seem harmless or unknown, the **target** of the command can be highly sensitive (e.g., `.zshrc`, `.env`). Additionally, there are other shell features that bypass simple binary classification.

Here is the plan to handle these edge cases robustly.

## Proposed Enhancements

### 1. Sensitive File Scanning in Shell Commands
Currently, `file:sensitive` only triggers when the agent uses the `writeFile` or `editFile` tools. If the agent runs `cat .env` or `rm ~/.zshrc` via the `shell` tool, it's just classified as `shell:unknown` or `shell:rm`.
**Solution:** We will introduce a fast substring scanner that checks the raw shell command for known sensitive strings.
If the command contains any of the following, it will immediately be flagged as `file:sensitive` (Destructive Tier):
- Project secrets: `.env`, `.pem`, `.key`, `id_rsa`, `id_ed25519`, `id_ecdsa`, `.git/config`, `.koincode/config.json`
- System/User configs: `.zshrc`, `.bashrc`, `.bash_profile`, `.bash_history`, `.ssh`, `/etc/passwd`, `/etc/shadow`, `~/.config`

### 2. Write Redirection Detection (`> ` and `>>`)
If the agent runs `echo "foo" > some-file.txt`, the command starts with `echo` (which is unmapped, so it defaults to `shell:unknown`). However, this is fundamentally a **write** operation and should be classified as `shell:write`.
**Solution:** We will add a parser that looks for `>` and `>>` operators outside of quotes. If a write redirection is found, the command's base classification will be elevated to `shell:write` (unless it's already something more destructive like `shell:rm` or `file:sensitive`).

### 3. Privilege Escalation Commands (`sudo`, `su`)
If the agent attempts to run `sudo rm -rf /`, the current map doesn't explicitly know `sudo`. It defaults to `shell:unknown` with a **normal** tier.
**Solution:** We will map `sudo` and `su` explicitly to the `destructive` tier so that any attempt to escalate privileges is treated with the highest caution.
